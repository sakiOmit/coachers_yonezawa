# ホームページ納品チェックリスト

このドキュメントは、ホームページ納品時の品質保証チェックリストです。

## 使い方

### 社内ダブルチェック用

すべての項目を確認し、チェックボックスにチェックを入れてください。確認者と確認日を記録表に記入します。

### クライアント提出用

このチェックリストをPDF化またはMarkdownで提出し、各項目がクリアされていることを証明します。

---

## 1. 技術品質チェック

### 1.1 コーディング規約準拠

- [ ] **FLOCSS + BEM命名規則に準拠**
  - 確認方法: SCSSファイルのクラス名が `.p-[page]__element` 形式になっているか確認
  - 参照: `docs/coding-guidelines/02-scss-design.md`

- [ ] **レスポンシブ対応（PC/タブレット/SP）**
  - 確認方法: Chrome DevToolsで各ブレークポイント（375px、768px、1024px、1440px）を確認
  - 確認ツール: Playwright MCP または Chrome DevTools MCP

- [ ] **クロスブラウザ対応**
  - 確認ブラウザ: Chrome、Safari、Firefox、Edge（各最新版）
  - 確認方法: 各ブラウザで全ページの表示・動作を確認

- [ ] **HTMLバリデーション**
  - 確認方法: [W3C Markup Validation Service](https://validator.w3.org/) でエラーがないことを確認
  - 許容範囲: WordPress自動生成タグによる軽微な警告は許容

- [ ] **CSSバリデーション**
  - 確認方法: [W3C CSS Validation Service](https://jigsaw.w3.org/css-validator/) で確認
  - 許容範囲: ベンダープレフィックスによる警告は許容

- [ ] **ケバブケース（kebab-case）の徹底**
  - 確認方法: すべてのBEMクラス名がケバブケースになっているか確認（キャメルケース禁止）
  - 正: `.p-page__main-visual` / 誤: `.p-page__mainVisual`

### 1.2 パフォーマンス

- [ ] **Core Web Vitals基準値クリア**
  - LCP (Largest Contentful Paint): 2.5秒以内
  - FID (First Input Delay): 100ms以内
  - CLS (Cumulative Layout Shift): 0.1以下
  - 確認ツール: Chrome DevTools MCP の Performance API、または [PageSpeed Insights](https://pagespeed.web.dev/)

- [ ] **画像最適化**
  - WebP形式に変換済み（対応ブラウザのみ）
  - Retina対応（2x画像生成）
  - `npm run image-opt` 実行済み
  - ファイルサイズが適切（目安: 200KB以下/枚）

- [ ] **Lazy Loading実装**
  - ファーストビュー画像: `loading="eager"` 設定
  - スクロール後の画像: `loading="lazy"` 設定
  - 確認方法: `render_responsive_image()` 関数が正しく使用されているか確認

- [ ] **CSS/JSファイル最小化**
  - 確認方法: `npm run build` 実行後、`themes/{{THEME_NAME}}/assets/` に minify されたファイルが生成されているか確認

- [ ] **キャッシュ設定**
  - ブラウザキャッシュ設定済み（.htaccess または nginx.conf）
  - 静的ファイル（CSS/JS/画像）のキャッシュ期間: 1年

### 1.3 アクセシビリティ

- [ ] **セマンティックHTML使用**
  - `<header>`, `<main>`, `<footer>`, `<nav>`, `<article>`, `<section>` 等を適切に使用
  - 見出しレベル（h1〜h6）が適切

- [ ] **WAI-ARIA属性の適切な使用**
  - ナビゲーションに `aria-label` または `aria-labelledby` 設定
  - ハンバーガーメニュー等に `aria-expanded` 設定

- [ ] **キーボード操作可能**
  - Tabキーで全リンク・ボタンにフォーカス可能
  - Enterキーでリンク・ボタンが動作
  - フォーカス時の視覚的フィードバックあり

- [ ] **alt属性設定**
  - すべての `<img>` タグに適切な `alt` 属性
  - 装飾画像は `alt=""` （空文字）
  - 意味のある画像は説明文を記載

- [ ] **カラーコントラスト比**
  - テキストと背景のコントラスト比: 4.5:1 以上（AA基準）
  - 確認ツール: Chrome DevTools の Lighthouse または [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/)

## 2. 機能・動作確認

### 2.1 ページ表示確認

- [ ] **全ページ正常表示**
  - トップページ（`/`）
  - ビジョン（`/vision/`）
  - 会社情報（`/company/`）
  - ABOUT US（`/aboutus/`）
  - メッセージ（`/message/`）
  - アートプロジェクト（`/art/`）
  - お問い合わせ（`/contact/`）
  - サンクスページ（`/thanks/`）
  - TVCMギャラリー（`/tvcm/`）
  - お知らせ一覧（`/news/`）
  - お知らせ詳細（`/news/:slug/`）
  - プライバシーポリシー（`/privacy/`、`/privacy-recruit/`、`/privacy-social-media/`）
  - 採用メッセージ（`/recruit-message/`）
  - 404ページ

- [ ] **ファーストビュー確認**
  - 各ページのファーストビューが正しく表示されるか
  - ヒーロー画像・キービジュアルが適切に表示されるか

- [ ] **ヘッダー・フッター確認**
  - ロゴ・ナビゲーションが正しく表示
  - フッター情報（住所・電話番号・著作権表記）が正確

### 2.2 リンク・ナビゲーション

- [ ] **内部リンク動作確認**
  - すべてのページ内リンクが正しく遷移
  - アンカーリンクが正しくスクロール
  - グローバルナビゲーションの現在地ハイライト

- [ ] **外部リンク動作確認**
  - 外部リンクに `target="_blank"` 設定
  - `rel="noopener noreferrer"` 設定済み
  - リンク先が正しいURLか確認

- [ ] **パンくずリスト動作**
  - 各下層ページでパンくずリストが正しく表示
  - リンクが正しく動作

- [ ] **ページネーション動作**
  - お知らせ一覧のページネーションが正しく動作
  - 次ページ・前ページリンクが正常

### 2.3 フォーム

- [ ] **ContactForm7動作確認**
  - お問い合わせフォームが正しく表示
  - 入力フィールドが正常に動作

- [ ] **入力バリデーション**
  - 必須項目の未入力でエラー表示
  - メールアドレス形式チェック
  - 電話番号形式チェック

- [ ] **送信完了ページ遷移**
  - 送信後、サンクスページ（`/thanks/`）に遷移
  - サンクスページが正しく表示

- [ ] **メール送信確認**
  - 管理者宛メールが届くか確認
  - 自動返信メールが届くか確認
  - メール本文の内容が正しいか確認

### 2.4 メディア・アニメーション

- [ ] **画像表示確認**
  - すべての画像が正しく表示
  - Retina対応画像が正しく読み込まれるか（2x環境で確認）
  - 画像の縦横比が崩れていないか

- [ ] **動画再生確認**
  - 動画が正しく再生されるか
  - 自動再生・ミュート設定が正しいか

- [ ] **アニメーション動作確認**
  - GSAP アニメーションが正しく動作（該当ページのみ）
  - スクロールトリガーアニメーションが正常
  - ホバーエフェクトが正しく動作

- [ ] **スムーススクロール動作**
  - ページ内アンカーリンクでスムーススクロール
  - ページトップへ戻るボタンが正常動作

## 3. SEO・メタ情報

### 3.1 メタタグ

- [ ] **各ページのtitle設定**
  - ページごとに適切な `<title>` タグ
  - 形式: `ページ名 | サイト名` （60文字以内推奨）
  - 確認方法: ブラウザのタブタイトル、またはページソース確認

- [ ] **meta description設定**
  - 各ページに適切な `<meta name="description">` タグ
  - 120〜160文字程度で要約
  - キーワードを自然に含める

- [ ] **OGP設定**
  - `og:title`: ページタイトル
  - `og:description`: ページ説明文
  - `og:image`: OGP画像（推奨サイズ: 1200x630px）
  - `og:url`: 正規URL
  - `og:type`: website または article
  - 確認ツール: [Facebook Sharing Debugger](https://developers.facebook.com/tools/debug/)

- [ ] **Twitter Card設定**
  - `twitter:card`: summary_large_image
  - `twitter:title`: ページタイトル
  - `twitter:description`: ページ説明文
  - `twitter:image`: Twitter Card画像
  - 確認ツール: [Twitter Card Validator](https://cards-dev.twitter.com/validator)

### 3.2 構造化データ

- [ ] **JSON-LD実装（必要に応じて）**
  - Organization（組織情報）
  - BreadcrumbList（パンくずリスト）
  - WebPage（ページ情報）
  - 確認ツール: [Google Rich Results Test](https://search.google.com/test/rich-results)

- [ ] **パンくずリスト構造化データ**
  - パンくずリストに構造化データを追加
  - JSON-LD形式で実装

### 3.3 検索エンジン対応

- [ ] **robots.txt設定**
  - `/robots.txt` が正しく配置されているか
  - クロール許可・禁止の設定が適切か
  - 確認方法: `https://example.com/robots.txt` にアクセス

- [ ] **XMLサイトマップ生成**
  - XMLサイトマップが生成されているか
  - すべてのページが含まれているか
  - 確認方法: `https://example.com/sitemap.xml` にアクセス

- [ ] **ファビコン設定**
  - ファビコン（favicon.ico）が設置されているか
  - Apple Touch Icon（apple-touch-icon.png）が設置されているか
  - サイズ: 16x16、32x32、180x180、192x192

- [ ] **Google Search Console登録**
  - サイトの所有権確認完了
  - XMLサイトマップ送信完了
  - インデックスカバレッジ確認

## 4. セキュリティ・運用準備

### 4.1 セキュリティ

- [ ] **HTTPS対応**
  - 全ページがHTTPSで配信されているか
  - HTTP→HTTPSへのリダイレクト設定済み
  - SSL証明書の有効期限確認

- [ ] **WordPress本体最新化**
  - WordPress本体が最新版か確認
  - 自動更新設定を確認

- [ ] **プラグイン最新化**
  - すべてのプラグインが最新版
  - 不要なプラグインを削除
  - 使用中プラグイン一覧を記録

- [ ] **不要ファイル削除**
  - サンプルページ（sample-page）削除
  - Hello Dolly プラグイン削除
  - 不要なテーマ削除
  - readme.html、license.txt 等の削除

- [ ] **セキュリティヘッダー設定**
  - X-Content-Type-Options: nosniff
  - X-Frame-Options: SAMEORIGIN
  - X-XSS-Protection: 1; mode=block
  - 確認ツール: [Security Headers](https://securityheaders.com/)

### 4.2 WordPress設定

- [ ] **パーマリンク設定確認**
  - パーマリンク構造が適切か確認
  - カスタム構造の場合、正しく動作しているか確認

- [ ] **メディア設定確認**
  - サムネイルサイズ設定が適切
  - 画像アップロード先が正しい

- [ ] **ユーザー権限設定**
  - 管理者アカウントのパスワード強度確認
  - 不要なユーザーアカウント削除
  - 各ユーザーの権限が適切か確認

- [ ] **投稿・固定ページ公開状態確認**
  - 下書き状態のページがないか確認
  - テストページが公開されていないか確認

### 4.3 運用準備

- [ ] **バックアップ設定**
  - 自動バックアッププラグイン設定済み
  - バックアップ頻度: 毎日または毎週
  - バックアップ保存先: 外部ストレージ
  - リストア手順の確認

- [ ] **Google Analytics設定**
  - GA4プロパティ作成済み
  - トラッキングコード設置済み
  - データ収集が正常に動作しているか確認
  - 目標・イベント設定（必要に応じて）

- [ ] **Google Tag Manager設定（必要に応じて）**
  - GTMコンテナ作成済み
  - トラッキングタグ設定済み
  - プレビューモードで動作確認

- [ ] **CMS操作マニュアル作成**
  - WordPress管理画面の基本操作
  - 投稿・固定ページの作成・編集方法
  - メディアアップロード方法
  - プラグイン設定方法

- [ ] **メンテナンス手順書作成**
  - WordPress本体・プラグイン更新手順
  - バックアップ・リストア手順
  - トラブルシューティング

## 5. コンテンツ最終確認

### 5.1 テキスト

- [ ] **誤字脱字チェック**
  - すべてのページのテキストを確認
  - 特に会社名・人名・固有名詞を注意深く確認

- [ ] **文法・表現チェック**
  - 敬語・丁寧語の統一
  - 表記ゆれの統一（例: Webサイト / ウェブサイト）

- [ ] **著作権表記確認**
  - フッターの著作権表記が正確
  - 使用画像・素材の著作権クリア

### 5.2 法的事項

- [ ] **プライバシーポリシー設置**
  - プライバシーポリシーページ（`/privacy/`）が設置されているか
  - 内容が最新の法令に準拠しているか
  - お問い合わせフォームからリンクされているか

- [ ] **特定商取引法表記（必要に応じて）**
  - ECサイト等の場合、特定商取引法に基づく表記が必要
  - 事業者名・住所・電話番号・返品条件等を記載

- [ ] **クッキーポリシー（必要に応じて）**
  - Cookie使用に関する通知・同意取得
  - EUユーザー向けの場合、GDPR対応

## 6. クライアント最終確認

- [ ] **デザイン最終確認**
  - クライアントからデザインカンプとの差異指摘がないか
  - 修正依頼がすべて反映されているか

- [ ] **コンテンツ最終確認**
  - クライアントからテキスト・画像の差し替え依頼がないか
  - 最終原稿がすべて反映されているか

- [ ] **機能動作確認**
  - クライアント自身で各機能をテスト
  - お問い合わせフォーム送信テスト

- [ ] **運用マニュアル確認**
  - CMS操作マニュアルがクライアントにとって理解しやすいか
  - 追加説明・修正依頼がないか

## 納品前最終チェック

- [ ] すべてのチェック項目が完了している
- [ ] 1次確認・2次確認（ダブルチェック）が完了している
- [ ] クライアントからの最終承認を得ている
- [ ] 納品物一式（ファイル・マニュアル）が揃っている
- [ ] 運用開始後のサポート体制を確認している
