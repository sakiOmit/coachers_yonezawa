# Next - 次のステップ案内

プロジェクトの現在の状態を分析し、次に何をすべきか自動で案内します。

## 使用方法

```bash
/next
```

## 実行内容

プロジェクトの状態を自動判定して、次のアクションを提案します。

### 判定ロジック

```
1. 最新の納品チェックリストを確認
   ├─ 存在しない → "/qa full を実行してください"
   ├─ コード修正課題あり → "/delivery-fix を実行してください"
   ├─ 手動確認項目あり → "/delivery-verify を実行してください"
   └─ すべて完了 → "/qa verify で最終確認してください"

2. QAレポートを確認
   ├─ 問題が検出された → "/qa fix を実行してください"
   └─ 問題なし → "開発継続またはデプロイ準備"

3. git statusを確認
   ├─ 未コミットの変更あり → "変更内容を確認してコミットしてください"
   └─ クリーン → "新規作業を開始できます"
```

## 実装ロジック

1. **最新の納品チェックリストを検索**
   ```bash
   ls -t reports/delivery-checklist-*.md 2>/dev/null | head -1
   ```

2. **チェックリストが存在する場合、進捗を解析**
   - `[ ]` と `[x]` をカウント
   - 優先度別に集計

3. **git statusを確認**
   ```bash
   git status --short
   ```

4. **状態に応じて次のアクションを提案**

## 連携コマンド

`/next` の提案に従って以下のコマンドを実行:
- `/qa full` - 完全QA実行
- `/delivery-fix` - コード修正課題を対話形式で修正
- `/delivery-verify` - 手動確認項目を対話形式で検証
- `/qa verify` - 最終検証
- `/delivery-report` - レポート更新

---

**このコマンドを実行するだけで、常に「次に何をすべきか」が分かります。**
