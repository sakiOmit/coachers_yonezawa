# セクション別実装戦略ガイド

Figma実装時に、**ページ規模に応じて最適な実装戦略を選択**するためのガイドです。

## 📏 ページ規模の判定基準

Figmaメタデータからセクション数を取得後、以下で判定:

### 小規模ページ（セクション数: 1〜3）

**特徴:**
- シンプルな構成
- 繰り返しパターンが少ない
- 総行数: ~100行以下

**例:**
- サンクスページ
- お問い合わせフォーム
- プライバシーポリシー

**推奨戦略: 一括実装**

```
Serena調査: 1回（ページ全体）
実装: まとめて実装
検証: セクション別スクリーンショット

トークン効率: 優 (~15,000 tokens)
```

---

### 中規模ページ（セクション数: 4〜6）

**特徴:**
- 明確な繰り返しパターン
- 総行数: 100〜200行

**例:**
- page-strength.php (5セクション)
- page-aboutus.php
- page-business.php

**推奨戦略: パターン理解 → 一括実装**

```
Step 1: Serenaでパターン調査（1回）
  → get_symbols_overview(page-strength.php)
  → 5つの.p-strength__itemが繰り返しと判明

Step 2: 1セクションの詳細調査（1回）
  → find_symbol("p-strength__item", depth=2)
  → パターン構造を完全理解

Step 3: 5セクション分まとめて実装
  → 繰り返しパターンを適用

Step 4: セクション別検証
  → Playwrightでセクション単位キャプチャ

トークン効率: 優 (~20,000 tokens)
コンテキスト管理: 問題なし
```

**🚫 避けるべき:** セクションごとにSerena調査（過剰）

---

### 大規模ページ（セクション数: 7以上、または200行以上）

**特徴:**
- 複雑な構成
- セクションごとに異なるレイアウト
- 総行数: 200行以上

**例:**
- page-sdgs.php (311行)
- page-vision.php (233行)
- トップページ（複数セクション）

**推奨戦略: セクション分割実装**

```
Step 1: ページ構造把握（1回のSerena）
  → get_symbols_overview(page-sdgs.php)
  → セクション一覧とパターン分類

Step 2: セクショングループ化
  例: SDGsページ
  - グループA: 同一パターン（セクション1〜3）
  - グループB: グリッド型（セクション4〜6）
  - グループC: アコーディオン型（セクション7〜9）

Step 3: グループごとにSerena調査・実装
  → find_symbol("p-sdgs__group-a", depth=2)
  → グループA実装
  → find_symbol("p-sdgs__group-b", depth=2)
  → グループB実装
  ...

Step 4: セクション別検証・統合

トークン効率: 良 (~35,000〜50,000 tokens)
コンテキスト管理: グループ単位で最適化
```

**メリット:**
- コンテキスト忘れ防止
- 複雑性の分割統治
- 並列実装が可能（複数人開発時）

```
Figmaメタデータ取得

セクション数カウント
  ↓
┌─────────────────────┐
│ セクション数: 1〜3   │ → 一括実装戦略

├─────────────────────┤

│ または 推定200行以上 │
└─────────────────────┘
```

## 💡 コンテキスト忘れ防止のベストプラクティス

### 1. Serenaメモリ活用

**大規模ページでは必須:**

```
Step 1: ページ構造をメモリ化
write_memory("page-sdgs-structure.md", セクション構造情報)

Step 2: セクション実装時に参照
read_memory("page-sdgs-structure.md")
→ 全体構造を思い出しながら個別実装

Step 3: 完了後にメモリ削除
delete_memory("page-sdgs-structure.md")
```

### 2. エージェント連携でコンテキスト保持

**wordpress-professional-engineerを段階的に使用:**

```
# ❌ 1回で全部やろうとする（コンテキスト忘れリスク）
Task tool: wordpress-professional-engineer
  → 10セクション全部実装して！

# ✅ グループごとに分割（コンテキスト保持）
Task tool: wordpress-professional-engineer
  → グループA（セクション1〜3）を実装

Task tool: wordpress-professional-engineer
  → グループB（セクション4〜6）を実装

Task tool: wordpress-professional-engineer
  → グループC（セクション7〜9）を実装
```

### 3. 繰り返しパターンの明示

**実装指示時に構造を明示:**

```
# ❌ 曖昧な指示
5つのセクションを実装してください

# ✅ 構造を明示
以下の繰り返しパターンで5セクション実装:

パターン:
.p-strength__item
  .p-strength__item-container
    .p-strength__item-content
      .p-strength__item-text
        .p-strength__item-number (01〜05)
        .p-strength__item-heading
        .p-strength__item-description
      .p-strength__item-image

変動部分:
- __item--01〜05 (Modifier)
- __item-content--reverse (偶数セクションのみ)
- テキスト・画像のみ変更
```

## 📊 戦略別トークン比較

| ページ規模 | 推奨戦略 | Serena調査回数 | 総トークン | コンテキスト管理 |
|-----------|---------|--------------|----------|--------------|
| 小規模（1〜3セクション） | 一括実装 | 1回 | ~15,000 | ✅ 問題なし |
| 中規模（4〜6セクション） | パターン理解→一括 | 2回 | ~20,000 | ✅ 問題なし |
| 大規模（7+セクション） | セクション分割 | 3〜5回 | ~35,000〜50,000 | ✅ グループ化で最適 |

---

## 🚀 `/figma-implement`への統合

このガイドは `/figma-implement` コマンドのStep 3（WordPress実装）で自動適用されます。

エージェントが以下を自動判定:
1. Figmaメタデータからセクション数を取得
2. 上記フローで戦略を選択
3. 最適なSerena調査・実装戦略を実行

**ユーザー側の操作: 不要**
